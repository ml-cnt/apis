    Authentication

    Basic setup

    
var jsonwebtoken = require('jsonwebtoken');
var api_key = "idnviKJKJIIaa"
var api_secret = "8e2d92f39c8f303ad8336d61f81bd06ee4e5d4..."
var exp = ((new Date()).getTime() / 1000) + 60;
var payload = {
  key: api_key,
  exp: exp
}
var jwt = jsonwebtoken.sign(payload, api_secret);

// use JWT in config
var config = {
  headers: {
    'Authorization': jwt
  }
};

// request using Axios
// GET
axios.get('https://dev-ticket.cloudandthing.com/api/v1/user/someapi', config).then(function(response){
    console.log(response)
  }); 

// POST,PUT,PATCH
var params = {};
axios.post('https://dev-ticket.cloudandthing.com/api/v1/user/someapi', params, config).then(function(response){
    console.log(response)
  });

    Authorization is JWT created from jsonwebtoken or any other libraries.

    CloudAndTHING Ticket uses JSON Web Tokens (JWT) to authorize all api requests. For more information, please read jwt.io

    The JWT needs to be encrypted with the HS256 algorithm (which should be noted in the token headers) and it must be signed with your API secret.

    The JWT payload required in the APIs are:

    • key - Api key
    • exp - Time.now + 1.minute (Expiry Time in epoch, Unix time in seconds).
    It is important that the expiry time must be in the Unix time format and no more than one minute in the future. Always regenerate JWT for every single API request.

    Tickets Report

    Created Tickets

    
axios.get('/api/v1/ticket_groups/SKU123456/tickets/created')
  .then(function (response) {
    console.log(response);
  })
  .catch(function (error) {
    console.log(error);
  });

    The above command returns JSON structured like this:

    [
  {
    "ticket_number": "12345",
    "status": 'created'
  },
  {
    "ticket_number": "12346",
    "status": 'created'
  }
]

    This endpoint retrieves all ticket with status created.

    HTTP Request

    GET /api/v1/ticket_groups/${sku}/tickets/created

    Query Parameters

    Parameter Type Description
    ticket_number String Ticket number
    status String ‘created’

    Errors

    This error section is stored in a separate file in `includes/_errors.md`. Slate allows you to optionally separate out your docs into many files…just save them to the `includes` folder and add them to the top of your `index.md`‘s frontmatter. Files are included in the order listed.

    The Kittn API uses the following error codes:

    Error Code Meaning
    400 Bad Request – Invalid Request
    401 Unauthorized – Your API key is wrong
    403 Forbidden – The kitten requested is hidden for administrators only
    404 Not Found – The specified kitten could not be found
    405 Method Not Allowed – You tried to access a kitten with an invalid method
    406 Not Acceptable – You requested a format that isn’t json
    410 Gone – The kitten requested has been removed from our servers
    418 I’m a teapot
    429 Too Many Requests – You’re requesting too many kittens! Slow down!
    500 Internal Server Error – We had a problem with our server. Try again later.
    503 Service Unavailable – We’re temporarily offline for maintenance. Please try again later.
