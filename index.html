    Authentication

    Basic setup

    
var jsonwebtoken = require('jsonwebtoken');
var api_key = "OPscp3BEng"
var api_secret = "XlnhM3yzCuNBZxup-4pCioifTbjmr_0bBA_L90qAf6bbvWtA7bDI2SmN4j8-E65C"
var exp = ((new Date()).getTime() / 1000) + 60;
var payload = {
  key: api_key,
  exp: exp
}
var jwt = jsonwebtoken.sign(payload, api_secret);

// use JWT in headers
var headers = {
  headers: {
    'Authorization': "Auth " + jwt
  }
};

// request using Axios
// GET
axios.get('https://dev-mellow-lounge-v2.cloudandthing.com/api/v2/user/someapi', headers).then(function(response){
    console.log(response)
  });

// POST,PUT,PATCH
var params = {};
axios.post('https://dev-mellow-lounge-v2.cloudandthing.com/api/v2/user/someapi', params, headers).then(function(response){
    console.log(response)
  });

    Authorization is JWT created from jsonwebtoken or any other libraries.

    CloudAndTHING Ticket uses JSON Web Tokens (JWT) to authorize all api requests. For more information, please read jwt.io

    The JWT needs to be encrypted with the HS256 algorithm (which should be noted in the token headers) and it must be signed with your API secret.

    The JWT payload required in the APIs are:

    • key - Api key
    • exp - Time.now + 1.minute (Expiry Time in epoch, Unix time in seconds).
    It is important that the expiry time must be in the Unix time format and no more than one minute in the future. Always regenerate JWT for every single API request.

    Member

    Member คือผู้ถือพาส และจะมีข้อมูลได้ตามที่กำหนดไว้ในหน้า Setting ของ Mellow Lounge ใน Menu ชื่อ Member Fields คุณสมบัติต่างๆที่สำคัญของ Member คือ

    1. Member จะต้องมี Member Group เสมอ
    2. membership_number เปรียบเสมือน Primary Key ห้ามซ้ำโดยเด็ดขาดทุก status
    3. Member จะมี Member UID ถูกสร้างขึ้นมาทุกครั้งโดยที่ Mellow Lounge เป็นผู้สร้างให้
    4. Status ของ Member มีสามแบบ โดยที่สามารถประยุกใช้ได้ตาม Use case ต่างๆ
      1. unverified - สร้าง Member ได้โดยที่ไม่มีการตรวจสอบ Field ห้ามซ้ำ
      2. registered - เปรียบเสมือน Member ที่ลงทะเบียนแล้ว ระบบจะบล๊อคข้อมูลห้ามซ้ำ
      3. verified - เปรียบเสมือน Member ที่ได้รับการตรวจสอบตัวตนแล้ว ระบบจะบล๊อคข้อมูลห้ามซ้ำ
    5. Fields ที่ห้ามซ้ำ
      1. telephone
      2. email
      3. identification_number

    Public link

    Public Link คือ API ที่ใช้สำหรับแจกจ่ายใน Public ซึ่งจะมีคุณสมบัติแตกต่างจาก create_pass_url ดังนี้

    1. Public Link จะต้องมีการเปิดใช้งาน Public Link ใน Member Group ก่อน
    2. Public Link จะต้องกรอกข้อมูลของผู้รับบัตร เพื่อยืนยันตัวตนก่อนที่จะรับบัตร โดยมี Fields ดังนี้
      1. Email
      2. Identification Number
      3. Running Number
      4. Membership Number
      5. First Name
      6. Last Name
      7. Telephone
      8. Day of Birth
      9. Month of Birth
      10. Year of Birth

    400 Bad Request – Invalid Request

    405 Method Not Allowed – You tried to access a kitten with an invalid method
    406 Not Acceptable – You requested a format that isn’t json
    410 Gone – The kitten requested has been removed from our servers
    418 I’m a teapot
    429 Too Many Requests – You’re requesting too many kittens! Slow down!
    500 Internal Server Error – We had a problem with our server. Try again later.
    503 Service Unavailable – We’re temporarily offline for maintenance. Please try again later.
